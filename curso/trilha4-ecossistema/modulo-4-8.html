üìà Scaling Strategies

AI tem caracteristicas unicas que afetam scaling: latencia variavel, dependencia de APIs externas, e custos por token.

Estrategias de Scaling

  • ‚Ä¢Auto-scaling: Scale baseado em request rate ou latencia
  • ‚Ä¢Rate limiting: Proteger de spikes e controlar custos
  • ‚Ä¢Queues: Buffer requests em picos (SQS, Redis)
  • ‚Ä¢Caching: Semantic cache para queries similares

Semantic Caching

# Cache baseado em similaridade semantica
# Queries similares retornam resultado cacheado

Query: "Qual a capital da Franca?"
Cache hit: "qual e a capital francesa" (0.95 similarity)
Result: "Paris" (from cache, 0ms)

üîí Seguranca em Producao

AI em producao tem vetores de ataque unicos: prompt injection, data leakage, e abuso de recursos.

‚ö†Ô∏è Ameacas Principais

  • ‚Ä¢Prompt Injection: Usuario manipula comportamento via input
  • ‚Ä¢Data Leakage: Modelo revela dados de treinamento/contexto
  • ‚Ä¢PII Exposure: Dados pessoais em logs ou outputs
  • ‚Ä¢Resource Abuse: Requests maliciosos para gastar tokens

‚úì Guardrails

  • ‚Ä¢ Input validation e sanitization
  • ‚Ä¢ Output filtering (PII, harmful content)
  • ‚Ä¢ Rate limiting por usuario
  • ‚Ä¢ Audit logging de todas as interacoes
  • ‚Ä¢ Separacao de ambientes (prod/staging)

üîÑ CI/CD para AI

Pipelines de CI/CD para AI precisam de testes especificos: evaluation gates, canary deploys, e rollback rapido.

Pipeline Exemplo

1Code push ‚Üí Lint + Unit tests
2Build ‚Üí Prompt validation
3Eval ‚Üí LLM evaluation on test dataset
4Gate ‚Üí Score > threshold?
5Deploy ‚Üí Canary (5%) ‚Üí Full rollout

üí° Prompt Versioning

Versione prompts como codigo. Cada mudanca de prompt e um "release" que deve passar por evaluation antes de ir para producao.
